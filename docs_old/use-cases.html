Built for Modern AI-Assisted Development

1. AI Agent UIs

Building agents that need to understand and generate UIs? Give them .uxm specs.

  • AI agents read component libraries natively
  • Understand relationships and composition
  • Generate implementations directly from specs
  • No vision models needed—perfect comprehension

Example: Agent reads button spec

{
  "id": "submit-button",
  "type": "button",
  "props": { "text": "Submit" },
  "accessibility": {
    "role": "button",
    "keyboardShortcuts": ["Enter", "Space"]
  }
}

Agent understands: This is a button. It says "Submit". It responds to Enter and Space keys. Perfect clarity—no interpretation needed.

3. Design Systems at Scale

Managing hundreds of component variations? Derivation model prevents duplication.

  • Change base components, all derivations inherit
  • Build libraries that grow organically through references
  • No more keeping 47 button copies in sync
  • Component graph scales without chaos

Example: Derivation vs Duplication

❌ Traditional (Duplication)

primary-button (copy)
large-button (copy)
small-button (copy)
danger-button (copy)

→ Change base style?
→ Update 4 files manually
→ Hope nothing breaks

✅ Fluxwing (Derivation)

primary-button (base)
  ↓ extends
large-button
small-button
danger-button

→ Change base once
→ All variations inherit
→ Single source of truth

4. Living Documentation

Design specs that both humans review and AI executes.

  • In version control, in PRs
  • Single source of truth
  • Always in sync—no documentation drift
  • Text-based diffs show actual design changes

Example: Design review in a PR

+ ## Hover State
+ ```
+ ╔══════════════════╗
+ ║   {{text}}       ║
+ ╚══════════════════╝
+ ```

- "border": "thin"
+ "border": "heavy"

Reviewers see: "Added hover state with heavy border". Visual change visible in git diff. No separate Figma link. No docs to update. The spec IS the docs.

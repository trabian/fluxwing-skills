How Skills Work

Understanding the agent invocation model, natural language activation, and task orchestration.

The Skills System

Fluxwing Skills is built on Claude Code's skills system - a way to extend Claude's capabilities through natural language-activated workflows.

How Skills Activate

  1. You say something to Claude: "Create a button"
  2. Claude analyzes your intent against installed skills
  3. Finds matching skill: uxscii-component-creator
  4. Loads skill's SKILL.md file with instructions
  5. Executes the workflow defined in the skill

SKILL.md Format

Each skill has a SKILL.md file with YAML frontmatter:

---
name: UXscii Component Creator
description: Create uxscii components with ASCII art and structured metadata
version: 1.0.0
author: Trabian
allowed-tools: Read, Write, Edit, Glob, Grep, Task, TodoWrite
---

# UXscii Component Creator

[Workflow instructions in markdown...]

The description field is used for matching against natural language triggers.

How Skills Invoke Agents

Skills use the Task tool to spawn agents with embedded instructions:

Task({
  subagent_type: "general-purpose",
  description: "Create uxscii component",
  prompt: `You are a uxscii component designer.

Component requirements:
- Name: ${componentName}
- Type: ${componentType}

Your task:
1. Load schema from {SKILL_ROOT}/schemas/uxm-component.schema.json
2. Create .uxm file (valid JSON)
3. Create .md file (ASCII template)
4. Save to ./fluxwing/components/

Follow uxscii standard strictly.`
})

The skill embeds all necessary context and instructions in the agent's prompt.

When to Orchestrate vs Execute

Approach Use When Example Skills
Orchestrate Complex, multi-step tasks requiring specialized work Component Creator, Screen Scaffolder, Screenshot Importer
Execute Simple, read-only, or direct file operations Library Browser, Component Viewer, Component Expander

Parallel Execution

The Power of Parallel Agents

Skills can spawn multiple agents in parallel for massive speedup:

Scenario Sequential Parallel Speedup
Create 6 components ~180 seconds ~30 seconds 6x faster
Vision analysis (3 tasks) ~45 seconds ~15 seconds 3x faster

How to Spawn Parallel Agents

Send a single message with multiple Task tool calls:

// CORRECT: Parallel execution
Task({ ... }) // Component 1
Task({ ... }) // Component 2
Task({ ... }) // Component 3

// WRONG: Sequential execution
TodoWrite([...])  // Creates list
for each todo:
  Task({ ... })   // One at a time
üöÄ Critical for performance: The Screen Scaffolder uses this pattern to create all missing components simultaneously instead of one-by-one.

Skill Workflow Patterns

Pattern 1: Single-Task Execution

Used by: Component Viewer, Library Browser

1. Parse user request
2. Read files directly
3. Display output
4. Done

Pattern 2: Single-Agent Orchestration

Used by: Component Creator (single component)

1. Parse user request
2. Gather requirements
3. Spawn designer agent
4. Wait for completion
5. Display result

Pattern 3: Multi-Agent Orchestration

Used by: Component Creator (multiple), Screen Scaffolder

1. Parse user request
2. Identify missing components
3. Spawn N designer agents in parallel
4. Wait for all completions
5. Spawn composer agent (if needed)
6. Display result

Pattern 4: Sequential Pipeline

Used by: Screenshot Importer

1. Parse screenshot path
2. Spawn vision coordinator (spawns 3 parallel vision agents)
3. Wait for vision analysis
4. Spawn N component generators in parallel
5. Spawn screen composer
6. Display result

Skills Communication

Between Skill and Agent

Skills communicate with agents through the Task tool's prompt:

Task({
  subagent_type: "general-purpose",
  description: "Create component",
  prompt: `Instructions here...

  Context:
  - Component name: ${name}
  - Required files: .uxm, .md
  - Output location: ./fluxwing/components/

  Steps to follow:
  1. Load schema
  2. Create files
  3. Validate`
})

Agent Response

Agents return results when complete:

{
  "success": true,
  "files_created": [
    "./fluxwing/components/submit-button.uxm",
    "./fluxwing/components/submit-button.md"
  ],
  "message": "Component created successfully"
}

Error Handling

Skills handle agent failures gracefully:

if (agent_result.success === false) {
  console.log(`Error: ${agent_result.error}`);
  // Retry logic or user notification
}

The Golden Rules

  1. Skills READ from {SKILL_ROOT}/templates/
  2. Skills WRITE to ./fluxwing/
  3. NEVER write to skill directories (read-only)
  4. NEVER read from workspace as templates (could be incomplete)

Performance Optimizations

Pre-Built Indexes

Library Browser uses pre-built template index:

// SLOW: Read 11+ files
const templates = glob('{SKILL_ROOT}/templates/*.uxm');
for (file of templates) {
  read(file); // 11+ file reads
}

// FAST: Read 1 index file
const index = read('{SKILL_ROOT}/data/template-index.json');
// All metadata instantly available!

Result: 10x faster browsing (1 file read vs 11+ file reads)

Modular Documentation

Skills load docs on-demand:

// Only load what's needed
if (creating_component) {
  load('{SKILL_ROOT}/docs/03-component-creation.md');
}
if (needs_ascii_patterns) {
  load('{SKILL_ROOT}/docs/06-ascii-patterns.md');
}

Each doc module: 500-800 tokens (saves context)

Parallel Agent Spawning

Spawn all independent agents in one message:

// ONE message with multiple Task calls
Task({ ... }); // Agent 1
Task({ ... }); // Agent 2
Task({ ... }); // Agent 3

// All execute in parallel automatically

Troubleshooting

Skill Not Activating

Problem: "Create a button" doesn't trigger the skill.

Solutions:

  • Check installation: ls ~/.claude/skills/uxscii-*/SKILL.md
  • Verify YAML frontmatter is valid
  • Try more explicit language: "Use the uxscii component creator"
  • Restart Claude Code

Agent Not Spawning

Problem: Skill runs but agent doesn't execute.

Solutions:

  • Check allowed-tools in YAML includes "Task"
  • Verify subagent_type is "general-purpose" (not custom type)
  • Check agent prompt has all required context

Slow Performance

Problem: Creating multiple components takes too long.

Solutions:

  • Ensure skill uses parallel Task calls (all in ONE message)
  • Check skill isn't using TodoWrite + sequential loops
  • Verify pre-built indexes are being used (Library Browser)

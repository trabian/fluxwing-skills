Quick Start

Get up and running with Fluxwing Skills in three simple steps:

1. Clone the Repository

git clone https://github.com/trabian/fluxwing-skills.git
cd fluxwing-skills

2. Run the Installer

./scripts/install.sh

The installer will automatically:

  • → Detect the best installation location (global or local)
  • → Copy all 6 skills to Claude Code
  • → Verify templates and schemas
  • → Show usage examples

3. Start Designing

Open Claude Code and say:

"Create a button component"

The uxscii-component-creator skill will activate and guide you through creating your first component with ASCII art and structured metadata.

─────────────────────────────────────────────────────────────────

Installation Options

Auto-Detect Mode (Recommended)

./scripts/install.sh

Logic: If .claude/ exists in current directory → Install locally (project-specific).
Otherwise → Install globally (~/.claude/skills/)

Global Installation

./scripts/install.sh --global

Install skills for all Claude Code sessions.
Location: ~/.claude/skills/

Local Installation

./scripts/install.sh --local

Install skills for current project only.
Location: ./.claude/skills/

Force Overwrite

./scripts/install.sh --force

Update existing skills without prompts.

Your First Component

Let's create a submit button using natural language:

Step 1: Ask Claude

You: "Create a submit button"

Step 2: Review the Output

Claude generates two files in ./fluxwing/components/:

submit-button.uxm (JSON metadata):

{
  "id": "submit-button",
  "type": "button",
  "version": "1.0.0",
  "metadata": {
    "name": "Submit Button",
    "created": "2024-10-11T12:00:00Z"
  },
  "props": {
    "text": "Submit"
  },
  "ascii": {
    "templateFile": "submit-button.md",
    "width": 20,
    "height": 3
  }
}

submit-button.md (ASCII template):

## Default State

```
╭──────────────────╮
│   {{text}}       │
╰──────────────────╯
```

Step 3: Iterate

Give feedback naturally:

You: "Make it bigger and add a hover state"

Claude: [Updates files with expanded dimensions and hover state]

Try These Next

Browse Available Components

"Show me all available components"

The uxscii-library-browser displays 11 bundled templates plus your custom components.

Add Interactive States

"Add hover and focus states to my button"

The uxscii-component-expander adds interaction states to existing components.

Build a Complete Screen

"Build a login screen with email, password, and submit button"

The uxscii-screen-scaffolder creates missing components and composes them into a complete screen.

View Component Details

"Show me the primary-button"

The uxscii-component-viewer displays full metadata and ASCII preview.

Import from Screenshots

"Import this screenshot: /path/to/design.png"

The uxscii-screenshot-importer uses vision analysis to extract components.

What Gets Installed

6 Specialized Skills

Skill Purpose
uxscii-component-creator Create UI components (buttons, inputs, cards)
uxscii-library-browser Browse available components and templates
uxscii-component-expander Add interaction states (hover, focus, disabled)
uxscii-screen-scaffolder Build complete screens from components
uxscii-component-viewer View component details and metadata
uxscii-screenshot-importer Import components from screenshot images

Supporting Files

  • → 11 Component Templates - Buttons, inputs, cards, modals, navigation, etc.
  • → 2 Screen Examples - Login screen and dashboard
  • → JSON Schema - Validation rules for component structure
  • → Documentation - 13 modular documentation files

Total: 6 skills + 42 supporting files = 48 files

Troubleshooting

Skills Not Activating

Problem: Natural language doesn't trigger skills.

Solutions:

  • → Verify installation: ls ~/.claude/skills/uxscii-*/SKILL.md
  • → Check YAML frontmatter: head -n 10 ~/.claude/skills/uxscii-component-creator/SKILL.md
  • → Restart Claude Code
  • → Try explicit language: "Use the uxscii component creator to make a button"

Templates Not Found

Problem: Skills can't find component templates.

Solutions:

  • → Verify templates: ls .claude/skills/uxscii-component-creator/templates/*.uxm
  • → Should show 11+ .uxm files
  • → Reinstall: ./scripts/install.sh --force

Permission Denied

Problem: Can't create skills directory.

Solutions:

  • → Check permissions: ls -la ~/.claude/
  • → Create manually: mkdir -p ~/.claude/skills
  • → Avoid sudo unless necessary

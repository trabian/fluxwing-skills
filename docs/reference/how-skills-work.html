How Skills Work

The Workflow

  1. You describe what you want in natural language
  2. Claude Code matches your request to a skill
  3. The skill activates with specialized instructions
  4. Component files (.uxm + .md) are generated in ./fluxwing/
  5. Files are validated against JSON Schema
  6. You can edit, version control, and reuse components

File Structure

./fluxwing/
├── components/          # Your components
│   ├── my-button.uxm    # JSON metadata
│   └── my-button.md     # ASCII template
├── screens/             # Complete screens
│   ├── login.uxm
│   ├── login.md
│   └── login.rendered.md
└── library/             # Customized templates

Key Principles

  • Read-only templates: Bundled templates in .claude/skills/ are read-only
  • Read-write workspace: All your work goes in ./fluxwing/
  • Schema validation: Components must pass JSON Schema validation
  • Progressive fidelity: Start simple (ASCII), add details (props, states)
